# 数据清洗与筛选逻辑说明

## 1. 输入数据结构

系统内部统一使用以下结构：

```json
{
  "id": "sample-001",
  "source": "demo",
  "question": "如何学习 Python 数据清洗？",
  "answer": "可以先学习 pandas、缺失值处理、重复值处理和异常值检测。",
  "metadata": {
    "url": "https://example.com"
  }
}
```

这样做的好处是：无论原始数据来自网页、CSV、JSONL 还是人工整理，都可以先转换为统一结构，后续处理模块不需要关心数据来源。

项目当前内置一批自建 demo 数据，说明见 `docs/dataset_design.md` 和 `data/raw/SOURCE_MANIFEST.md`。

## 2. 清洗规则

| 规则 | 目的 |
| --- | --- |
| 去除 HTML 标签 | 清理网页采集残留内容 |
| 反转义 HTML 实体 | 将 `&amp;` 等内容恢复为正常文本 |
| 统一中英文空白 | 避免换行、制表符影响去重 |
| 统一常见标点 | 降低格式差异导致的重复漏检 |
| 删除首尾空格 | 保证文本规范 |
| 过滤空问题或空答案 | 确保样本可用于训练 |
| 过滤过短答案 | 避免无信息量样本 |

## 3. 去重逻辑

项目使用两层去重：

1. 精确去重：对清洗后的问题和答案组合生成规范化 key。
2. 近似去重：使用字符 n-gram Jaccard 相似度判断文本是否高度相似。

近似去重虽然不是深度语义模型，但可以在无额外依赖的环境中稳定运行。后续如果接入 `sentence-transformers`，可以升级为向量相似度去重。

## 4. 质量评分维度

满分为 100 分，按以下规则扣分：

| 维度 | 扣分原因 |
| --- | --- |
| 答案太短 | 信息量不足 |
| 问题太短 | 语义不明确 |
| 广告或营销词 | 训练污染风险 |
| 异常字符过多 | 可能是乱码或噪声 |
| 问答主题不匹配 | 问题和答案关联弱 |
| 答案过长 | 可能混入正文或无关内容 |

评分结果分为：

- `high`: 80 分及以上，可优先进入训练集。
- `medium`: 60 到 79 分，建议人工抽检。
- `low`: 60 分以下，默认不导出到训练集。

## 5. 分类逻辑

当前版本使用关键词规则分类，类别包括：

- `tech`: Python、脚本、爬虫、工程、日志等。
- `data`: 清洗、去重、质量、数据集、pipeline 等。
- `ai`: 模型、训练、微调、分类、向量等。
- `career`: 简历、面试、岗位、实习等。
- `general`: 无明显关键词的通用样本。

这个设计是为了演示“数据流可控”。如果后续接入开源模型，可以将规则分类替换为模型分类。

## 6. 导出格式

导出文件为 `instruction_dataset.jsonl`，每行格式如下：

```json
{
  "instruction": "用户问题",
  "input": "",
  "output": "高质量答案",
  "category": "data",
  "quality_score": 92,
  "source": "demo"
}
```

该格式可以直接作为大模型微调数据的基础格式。
